# تقرير تحسين وتنسيق مقترح الصيانة المركزية للطباعة وتصدير PDF
# Print-First & PDF Layout Optimization Report

تمت إعادة تقييم تصميم وتنسيق موقع `saqala-proposal` بناءً على التوضيح الجديد: **المخرج النهائي للمشروع هو ملف مطبوع (ورقي أو PDF ثابت) للتقديم للإدارة العليا، دون الحاجة لأي تفاعل رقمي**. 

استخدام تقنيات الويب (HTML/CSS) لتوليد مستندات مطبوعة هو بديل قوي لبرنامج MS Word، ولكنه يتطلب ضبطاً دقيقاً لأنماط الطباعة (`@media print`) للتأكد من أن المستند يظهر بشكل مثالي وخالٍ من العيوب عند التصدير. يركز هذا التقرير على تحسينات التنسيق المطبوع (A4) وتلافي عيوب توزيع الصفحات.

---

## 1. التحكم بصفحات التقرير وفواصل الصفحات (Page Breaks & Margins)

التحكم الفائق بمكان بدء وانتهاء الصفحات هو أهم عامل لجعل الملف يبدو كأنه صُمم في مطبعة محترفة:

### أ. تنظيم فواصل الصفحات للمكونات (Avoid Page Splitting)
* **المشكلة:** عند طباعة الموقع حالياً، قد تنقطع بعض العناصر (مثل جداول البيانات، أو المخططات التدفئية، أو الصناديق التوضيحية) في المنتصف، لتظهر نصفها في أسفل الصفحة والنصف الآخر في الصفحة التالية.
* **الحل:** تطبيق خاصية منع الانقسام في ملف التنسيق `globals.css`:
  ```css
  @media print {
    /* منع انقسام الجداول، الصناديق، والمخططات */
    table, tr, td, th, .no-break, svg, .bg-white {
      break-inside: avoid !important;
      page-break-inside: avoid !important;
    }
  }
  ```

### ب. بدء الأقسام في صفحات جديدة (Force Page Break Before Sections)
* **المشكلة:** يضم المقترح 10 أقسام رئيسية. إذا انتهى القسم الأول في منتصف الصفحة، يبدأ القسم الثاني مباشرة في نفس الصفحة، وهو أسلوب غير احترافي في التقارير الإدارية.
* **الحل:** إجبار كل قسم رئيسي (Section) على البدء في صفحة جديدة دائماً عبر تغيير فئة الفواصل في `page.tsx` لتكون صارمة:
  ```css
  .page-break-section {
    break-before: page !important;
    page-break-before: always !important;
  }
  ```

### ج. إعداد هوامش ورقة الطباعة وهامش غلاف التقرير (Page Margin Management)
* **المشكلة:** تم تعيين الهامش العام للطباعة في `globals.css` بـ `margin: 16mm 18mm`. هذا الهامش يشوه الغلاف (`CoverPage.tsx`) لأنه يدفع العناصر الزخرفية (مثل الأشرطة والدوائر الزرقاء) بعيداً عن حواف الورقة، مما يمنع تصميم الغلاف الكامل (Full-bleed).
* **الحل:** تخصيص هوامش مستقلة للغلاف عن باقي الصفحات في CSS:
  ```css
  /* الهامش الافتراضي لصفحات التقرير */
  @page {
    size: A4;
    margin: 20mm 18mm 20mm 18mm; /* هوامش متزنة للكتابة */
  }

  /* إلغاء الهوامش للغلاف ليمتد التصميم لكامل الورقة */
  @page :first {
    margin: 0;
  }
  ```

## 2. تصحيح أخطاء اتجاه القراءة العربي (RTL Layout Alignments)

عند طباعة التقرير باللغة العربية، هناك أخطاء برمجية في محاذاة العناصر تظهر بوضوح على الورق المطبوع وتسيء للمظهر العام:

1. **صناديق التنبيهات والأهداف (`CallOutBox.tsx`):**
   * **المشكلة:** الحافة الملونة المميزة تظهر على اليسار `border-l-4`.
   * **الحل:** يجب تحويلها لليمين `border-r-4` لتتوافق مع بداية سطر القراءة العربي.

2. **خطوط الخطط الزمنية (`ProcessStep.tsx`):**
   * **المشكلة:** الخط العمودي الرابط بين الخطوات يظهر في أقصى اليسار `left-[19px]` بينما أرقام الخطوات تظهر في اليمين، مما يفصل الخط تماماً عن الأرقام على الورقة المطبوعة.
   * **الحل:** تحويل موضع الخط ليكون محاذياً ليمين المكون `right-[19px]` أو `start-[19px]`.

3. **عناوين الجداول (`DataTable.tsx`):**
   * **المشكلة:** رؤوس الجداول `th` تتجه لليسا `text-left` بينما البيانات تتجه لليمين، مما يظهر الأعمدة بشكل متداخل أو مشتت.
   * **الحل:** تحويل محاذاة الرؤوس إلى اليمين `text-right`.

4. **المخططات التدفئية (`FlowChart.tsx`):**
   * **المشكلة:** تتدفق أفقياً من اليسار لليمين (LTR)، وهو عكس حركة عين القارئ العربي.
   * **الحل:** تعديل كود رسم المخطط ليعكس التدفق ليكون من اليمين لليسار (RTL)، أو تحويل المخططات لتكون عمودية (TD) بالكامل لتناسب القراءة الورقية لأسفل.

## 3. تحسين مقروئية المخططات والرسومات على الورق (Chart Legibility)

على الورق المطبوع، لا يستطيع القارئ تكبير الشاشة أو تمريرها لرؤية النصوص الطويلة المقطوعة:

1. **التفاف النصوص داخل عقد المخطط (Text Wrapping in SVGs):**
   * **المشكلة:** نصوص المخططات الحالية داخل عناصر الـ SVG لا تلتف تلقائياً للسطر التالي إذا كانت طويلة، مما يؤدي لقص العبارات الفنية الهامة في النسخة المطبوعة.
   * **الحل:** تقليل طول العبارات داخل المخططات يدوياً في ملفات البيانات، أو برمجة الـ SVG لتوزيع النص على أسطر متعددة باستخدام `<tspan>` بدلاً من سطر واحد طويل.

2. **أبعاد المخططات الأفقية:**
   * **المشكلة:** المخططات العريضة جداً ستتقلص تلقائياً لتناسب عرض ورقة A4، مما يجعل نصوصها صغيرة جداً وغير مقروءة عند الطباعة.
   * **الحل:** تجنب المخططات الأفقية الطويلة (عريضة الأعمدة) واستبدالها بمخططات رأسية (عمودية) لتأخذ كامل طول الصفحة وتتم قراءتها بوضوح.

## 4. الخطوط وتناسق الألوان المطبوعة (Print Typography & Colors)

طباعة الألوان على الورق تختلف عن عرضها على الشاشات:

1. **الخط العربي الاحترافي (Executive Font):**
   * **المشكلة:** يعتمد الموقع على خط النظام الافتراضي (Segoe UI على ويندوز)، وهو خط يبدو جافاً وغير مناسب للمقترحات الفنية الفاخرة.
   * **الحل:** دمج خط **Cairo** أو **Tajawal** بوزن متوسط ورسم واضح، مما يمنح التقرير المطبوع مظهراً تنفيذياً فخماً وسهلاً للقراءة بوضوح.

2. **تباين الألوان واقتصادية الحبر (Contrast & Ink Economy):**
   * **المشكلة:** الخلفيات الملونة الفاتحة جداً (مثل الأصفر الخفيف للأمان أو الأحمر الخفيف للخطر) قد تظهر باهتة جداً أو رمادية مشوهة إذا طُبع الملف بطابعة أبيض وأسود أو طابعة مكتبية عادية.
   * **الحل:** التأكد من إرفاق إطار داكن وواضح (`border-2 border-[#...]`) لجميع صناديق التنبيهات لتعريف حدودها حتى لو طُبعت بدون ألوان، والاعتماد على نصوص داكنة عالية التباين (مثل الأسود الداكن والرمادي الغامق).

## 5. الترويسة والتذييل المتكرر وأرقام الصفحات (Headers, Footers & Page Numbers)

في الملفات المطبوعة الطويلة، يجب أن يظهر رقم الصفحة وعنوان التقرير في كل ورقة لتسهيل المرجعية:

* **المشكلة:** يحتوي التذييل الحالي على كود فارغ لأرقام الصفحات ولا يعمل ديناميكياً في الطباعة.
* **الحل:** تفعيل عداد الصفحات التلقائي في CSS للطباعة، لتقوم الطابعة باحتساب وترقيم الأوراق تلقائياً في الأسفل:
  ```css
  @media print {
    /* تعريف عداد الصفحات */
    body {
      counter-reset: page;
    }
    
    /* طباعة رقم الصفحة ديناميكياً في التذييل */
    .page-number::after {
      content: "صفحة " counter(page);
    }
  }
  ```
* **تأمين مساحة التذييل:** ضبط الهامش السفلي للورقة ليكون كافياً بحيث لا تتداخل بيانات التقرير مع التذييل المطبوع المتكرر.

## 6. ملخص تحويل كود الويب لخدمة الملف المطبوع (Print Tuning Summary)

| المكون الفني | التنسيق الحالي (تنسيق شاشة) | التنسيق المطلوب (تنسيق طباعة A4) | الفائدة للتقرير المطبوع |
| :--- | :--- | :--- | :--- |
| **فاصل الصفحات** | انسيابي مستمر مع خطوط زرقاء فاصلة | فاصل صارم يبدأ كل قسم في صفحة جديدة | تنظيم المستند وتوزيع الأفكار في أوراق مستقلة |
| **هوامش الغلاف** | هوامش بيضاء تحيط بزخارف الغلاف | هوامش صفرية للغلاف (Full-bleed) | مظهر غلاف احترافي يغطي كامل الورقة كالمجلات |
| **مكون المخطط** | تدفق أيسر (LTR) عريض | تدفق رأسي (Top-to-Bottom) أو عربي (RTL) | مخططات مقروءة دون الحاجة لتقليص حجم الخط |
| **توزيع المكونات** | انقسام عشوائي عند نهاية الصفحة | منع الانقسام `break-inside: avoid` | بقاء الجداول والبطاقات مكتملة في صفحة واحدة |
| **ترقيم الصفحات** | ترميز يدوي غير فعال | ترقيم ديناميكي للطابعة `counter(page)` | تسهيل الفهرسة ومراجعة الأقسام مع الإدارة |
| **الخطوط** | خط نظام افتراضي جاف | خط **Tajawal** أو **Cairo** الفاخر | إبراز جودة العمل والاهتمام بالهوية البصرية للشركة |
